# Quotes & B2B — Quote Request Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement a B2B quote-request workflow: companies, company memberships, a quote cart, quote request lifecycle (PENDING → ASSIGNED → DRAFT → SENT → ACCEPTED/REJECTED/EXPIRED/CANCELLED), PDF generation, email notifications, and checkout integration for accepted quotes.

**Architecture:** Two new packages (`b2b`, `quote`) following the existing package-by-feature layout. `CompanyService` owns company and membership management. `QuoteCartService` manages the per-user active quote cart. `QuoteService` owns the full quote request lifecycle, coordinates with `OrderService` and `PaymentService` on accept, and delegates PDF rendering to `QuotePdfService`. Domains talk only through service interfaces, never repository-to-repository.

**Tech Stack:** Spring Boot 4.0.4 · Java 25 · iTextPDF (PDF generation) · PostgreSQL (Flyway V15–V17) · Testcontainers

**Files:**
- Create: `src/main/resources/templates/quote-template.html` — Thymeleaf HTML template with company name, quote ID, items table (description, qty, unit price, line total), subtotal, tax note ("Taxes will be calculated at checkout"), delivery address, expiry date, staff notes.
- Create: `quote/service/QuotePdfService.java`
- Modify: `blob/service/StorageService.java` — add `InputStream fetch(String key)`
- Modify: `blob/service/S3StorageService.java` — implement `fetch` via `GetObjectRequest`

**QuotePdfService:**
- `generate(quote, items, company)` — renders Thymeleaf template to HTML string → converts to PDF bytes via OpenHTMLtoPDF `PdfRendererBuilder`.
- Returns `byte[]`.

Used by `QuoteService.send()` to produce the PDF before storing to blob storage and attaching `pdfBlobId` on the quote.

**PDF download — `QuoteService.downloadPdf(quoteId, userId)`:**
- Ownership check (ForbiddenException if mismatch).
- Guard: `pdfBlobId == null` → `NotFoundException("PDF_NOT_AVAILABLE")`.
- Load `BlobObject` by `pdfBlobId`; fetch bytes from `StorageService.fetch(blob.getKey())`.
- Returns `byte[]`.

**`GET /api/v1/quotes/{id}/pdf`** on `QuoteController`:
- Calls `quoteService.downloadPdf(id, user.getId())`.
- Returns `ResponseEntity<byte[]>` with `Content-Type: application/pdf` and `Content-Disposition: attachment; filename="quote-{id}.pdf"`.

## Task 14: Stripe Tax

**Goal:** Enable Stripe Tax on quote-originated Checkout Sessions so taxes are calculated and collected by Stripe at the point of payment. The quote PDF shows pre-tax subtotal only.

**Files:**
- Modify: `src/main/java/io/k2dv/garden/payment/service/PaymentService.java`
- Modify: `src/main/resources/templates/quote-template.html`

**`PaymentService.createCheckoutSessionFromQuote()`:**

Add `automatic_tax` and customer delivery address to the Stripe Checkout Session parameters:

```java
SessionCreateParams params = SessionCreateParams.builder()
    // ... existing line items, mode, urls, metadata ...
    .setAutomaticTax(SessionCreateParams.AutomaticTax.builder()
        .setEnabled(true)
        .build())
    .setCustomerDetails(SessionCreateParams.CustomerDetails.builder()
        .setAddress(SessionCreateParams.CustomerDetails.Address.builder()
            .setLine1(quote.getDeliveryAddressLine1())
            .setLine2(quote.getDeliveryAddressLine2())
            .setCity(quote.getDeliveryCity())
            .setState(quote.getDeliveryState())
            .setPostalCode(quote.getDeliveryPostalCode())
            .setCountry(quote.getDeliveryCountry())
            .build())
        .setAddressSource(SessionCreateParams.CustomerDetails.AddressSource.SHIPPING)
        .build())
    .build();
```

**`quote-template.html`:**

Replace the single "Grand Total" row with two rows — subtotal and a tax note:

```html
<tr class="subtotal-row">
  <td colspan="3">Subtotal</td>
  <td th:text="${#numbers.formatDecimal(grandTotal, 1, 2)}">0.00</td>
</tr>
<tr class="tax-note-row">
  <td colspan="4" style="font-style: italic; color: #7f8c8d;">
    Taxes will be calculated at checkout
  </td>
</tr>
```

**No new migration required** — tax is computed and collected by Stripe; no tax fields are added to the quote schema.

**Note on `Order.taxAmount`:** The Stripe webhook `checkout.session.completed` event includes `total_details.amount_tax`. Storing this on `Order` for audit purposes is a separate task and out of scope here.

## State Machine Summary

```
QuoteCart:    ACTIVE → SUBMITTED

QuoteRequest: PENDING
                └─ assign      → ASSIGNED
                     └─ draft items → DRAFT
                          └─ send   → SENT
                               ├─ accept  → ACCEPTED (creates Order + Stripe session)
                               │               └─ Stripe webhook → PAID (Order also PAID)
                               ├─ reject  → REJECTED  (customer)
                               └─ expire  → EXPIRED   (admin / lazy on accept)
              PENDING/ASSIGNED/DRAFT/SENT
                └─ cancel → CANCELLED  (staff or customer)
```
